# Discovery Documents

> All discovery documents for product ideas

## Purpose

This folder contains all discovery documents generated during the AI-powered discovery phase. Each idea has a complete set of discovery documents with consistent naming.

## Document Types

### Required Discovery Documents

1. **NARRATIVE-<idea-slug>.md**
   - Raw Manus narrative output
   - Niche insights, cultural context, worldview
   - Generated by: `@Manus-Narrative-Agent`

2. **NICHE-INTEL-<idea-slug>.md**
   - Niche size, psychographics, subcultures
   - Language patterns, identity signals, status markers
   - Communities and entry points
   - Generated by: `@Niche-Intelligence-Agent`

3. **PAIN-SIGNALS-<idea-slug>.md**
   - Pain clusters (frequency, severity, urgency)
   - Top pain signals ranked by score
   - Moment of Acute Pain (MAAP) analysis
   - Generated by: `@Pain-Signal-Agent`

4. **JTBD-<idea-slug>.md**
   - Deep persona narrative
   - Jobs-to-be-Done (main, related, emotional)
   - Triggers, success criteria, barriers
   - ElevenLabs voice scripts
   - Generated by: `@JTBD-Agent`

5. **OPPORTUNITY-<idea-slug>.md**
   - Opportunity & Moat analysis
   - Winner-take-most dynamics
   - Switching costs, data moats, workflow embedding
   - Opportunity Score (0-10)
   - Generated by: `@Opportunity-Moat-Agent`

### Optional Discovery Documents

6. **REDTEAM-<idea-slug>.md**
   - Critical analysis and red-teaming
   - False assumptions, competitor risks, moat stress test
   - Pivot recommendations
   - Generated by: `@Red-Team-Strategist`

## File Naming Convention

**Format**: `<DOC-TYPE>-<idea-slug>.md`

**Rules**:
- Document type in UPPERCASE (e.g., `NICHE-INTEL`, `PAIN-SIGNALS`)
- Idea slug in lowercase, hyphenated (e.g., `enterprise-design-system-startups`)
- All documents for the same idea use the same `<idea-slug>`

**Examples**:
- `/docs/discovery/NICHE-INTEL-enterprise-design-system-startups.md`
- `/docs/discovery/PAIN-SIGNALS-enterprise-design-system-startups.md`
- `/docs/discovery/JTBD-enterprise-design-system-startups.md`
- `/docs/discovery/OPPORTUNITY-enterprise-design-system-startups.md`
- `/docs/discovery/REDTEAM-enterprise-design-system-startups.md`

## Idea Slug Requirements

**Idea slugs must be**:
- Lowercase
- Hyphenated (no spaces, underscores, or special characters)
- Descriptive but concise
- Consistent across all documents for the same idea

**Examples**:
- ✅ `enterprise-design-system-startups`
- ✅ `real-estate-investor-flipper-platform`
- ✅ `ai-code-review-documentation-tool`
- ❌ `Enterprise Design System` (uppercase, spaces)
- ❌ `enterprise_design_system` (underscores)
- ❌ `eds` (too short, not descriptive)

## Complete Discovery Doc Set

**A complete discovery doc set for an idea includes**:
1. NARRATIVE (from Manus)
2. NICHE-INTEL
3. PAIN-SIGNALS
4. JTBD
5. OPPORTUNITY
6. REDTEAM (optional but recommended)

**All documents must use the same `<idea-slug>`.**

## Workflow

See `.cursor/rules/302-discovery-workflow.mdc` for the complete 9-step discovery workflow.

**Discovery Phase**:
1. Manus generates NARRATIVE
2. ChatGPT refines insights
3. Discovery agents produce NICHE-INTEL, PAIN-SIGNALS, JTBD, OPPORTUNITY
4. Red-Team Strategist produces REDTEAM (optional)
5. Opportunity Score calculated (must be ≥ 8.0/10 to proceed)

**After Discovery**:
- If Opportunity Score ≥ 8.0/10 → Proceed to Validation Phase
- If Opportunity Score < 8.0/10 → Pivot or Kill
