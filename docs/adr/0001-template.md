# ADR-0001: ADR Template

## Status

Accepted

## Context

We need a consistent format for documenting architectural decisions.

## Decision

Use this template for all ADRs:

```markdown
# ADR-XXXX: [Title]

## Status

[Proposed | Accepted | Deprecated | Superseded]

## Context

[Describe the issue motivating this decision]

## Decision

[Describe the change that we're proposing or have agreed to implement]

## Consequences

### Positive
- [Benefit 1]
- [Benefit 2]

### Negative
- [Drawback 1]
- [Drawback 2]

### Risks
- [Risk 1]
- [Risk 2]

## Alternatives Considered

- [Alternative 1]: [Why it was rejected]
- [Alternative 2]: [Why it was rejected]
```

## Consequences

### Positive
- Consistent documentation format
- Clear decision history

### Negative
- Requires discipline to maintain

### Risks
- ADRs may become outdated if not maintained
